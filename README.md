# Indus Valley Script Research Project

This workspace is now the project lab for a disciplined attempt to study the Indus Valley sign system.

The aim is not to announce a decipherment early. The aim is to build the strongest possible evidence pipeline: corpus hygiene, sign statistics, positional grammar, non-linguistic baselines, and only then constrained linguistic hypotheses.

## Working Stance

- Treat the Indus system as undeciphered.
- Treat all proposed readings as hypotheses, not data.
- Prefer reproducible tests over attractive resemblances.
- Compare Dravidian, Indo-Aryan/Indo-Iranian, Munda-like, isolate, and non-linguistic models under the same evidentiary rules.
- Separate graphemic questions from linguistic questions: first ask what the signs are doing, then ask whether speech is involved.

## Current Data

- `data/ivs_corpus_cleaned.csv`: initial neutral working corpus.
- `data/ivs_corpus.csv`: same row count, but includes proposed Sanskrit readings and translations. These fields are excluded from the baseline because they risk circular reasoning.

## Research Phases

1. Corpus audit and provenance.
2. Sign inventory normalization.
3. Directionality and positional class analysis.
4. Formula, n-gram, and network analysis.
5. Non-linguistic baseline modeling.
6. Linguistic hypothesis testing.
7. Narrow, falsifiable candidate readings only where distributional evidence permits them.

## Evidence Ladder

- `Speculative`: plausible idea, not yet tested.
- `Suggestive`: agrees with some observed pattern but has weak alternatives.
- `Testable`: predicts a measurable corpus pattern.
- `Statistically supported`: survives predefined quantitative tests.
- `Philologically plausible`: agrees with linguistic, archaeological, and comparative constraints.

No claim enters the project as a conclusion. It enters as a hypothesis and earns its place.
